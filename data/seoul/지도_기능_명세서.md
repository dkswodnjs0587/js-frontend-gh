# 지도 기능 명세서

## 1. 개요

서울 지역 JSON 데이터를 지도에 표시하는 단일 페이지 프로토타입이다. `mapx` / `mapy` 좌표를 기반으로 각 장소를 핀으로 보여주고, 검색과 유형 필터로 탐색할 수 있어야 한다.

## 2. 목적

- 서울 데이터에 포함된 모든 좌표를 지도에서 한눈에 확인한다.
- 유형별 데이터 분포를 시각적으로 파악한다.
- 특정 장소를 빠르게 검색하고 지도 위치로 이동한다.

## 3. 데이터 범위

- 대상 파일: `서울_관광지.json`, `서울_문화시설.json`, `서울_축제공연행사.json`, `서울_여행코스.json`, `서울_레포츠.json`, `서울_숙박.json`, `서울_쇼핑.json`
- 좌표 필드: `mapx`(경도), `mapy`(위도)
- 필수 표시 정보: 장소명 `title`, 주소 `addr1` 또는 `addr2`, 연락처 `tel`, 유형명

## 4. 화면 구성

### 4.1 상단 영역

- 서비스명, 간단한 설명, 로딩 상태 요약을 표시한다.
- 전체 로드된 파일 수, 좌표 수, 현재 표시 수를 요약 칩으로 보여준다.

### 4.2 좌측 사이드바

- 검색 입력창: 장소명, 주소, 전화번호 기준으로 검색한다.
- 유형 필터: 전체, 관광지, 문화시설, 축제공연행사, 여행코스, 레포츠, 숙박, 쇼핑을 선택한다.
- 통계 카드: 활성 파일 수, 좌표 포함 건수, 현재 표시 수, 검색 결과 수를 보여준다.
- 장소 목록: 현재 필터 결과 중 상위 24개를 카드로 표시한다.
- 폴백 로드: 브라우저 자동 fetch가 실패할 경우 JSON 폴더를 직접 선택해 다시 읽는다.

### 4.3 지도 영역

- 서울 중심 좌표를 기본 뷰로 표시한다.
- 로드된 장소를 마커로 표시한다.
- 같은 지역에 많은 마커가 겹치면 클러스터로 묶어 표시한다.
- 유형별 색상을 다르게 하여 분류를 즉시 인지할 수 있게 한다.
- 마커 클릭 시 팝업으로 상세 정보가 열린다.

## 5. 기능 요구사항

### 5.1 데이터 로딩

- 페이지 진입 시 현재 폴더의 서울 JSON 파일을 자동으로 읽는다.
- 각 JSON의 `items` 배열을 순회해 좌표가 있는 항목만 지도에 반영한다.
- 자동 로드가 실패하면 사용자에게 실패 사유를 보여주고 폴백 로드를 유도한다.

### 5.2 마커 표시

- 각 장소는 1개의 마커로 표시한다.
- 좌표가 없거나 숫자로 변환할 수 없으면 지도에 표시하지 않는다.
- 마커는 유형별 색상을 사용한다.
- 많은 데이터는 클러스터링으로 겹침을 완화한다.

### 5.3 검색

- 검색어는 대소문자와 무관하게 처리한다.
- 검색 대상은 `title`, `addr1`, `addr2`, `tel`, 유형명이다.
- 검색 결과가 변경되면 지도, 목록, 통계가 동시에 갱신된다.

### 5.4 유형 필터

- 전체 또는 개별 콘텐츠 유형을 선택할 수 있다.
- 필터 변경 시 검색 조건과 함께 다시 계산한다.
- 선택된 유형은 활성 상태로 시각적으로 강조한다.

### 5.5 목록 연동

- 지도에 보이는 장소 중 상위 24개를 목록에 보여준다.
- 목록 항목 클릭 시 해당 마커 위치로 이동하고 팝업을 연다.
- 목록이 비면 빈 상태 메시지를 표시한다.

### 5.6 지도 조작

- 기본 진입 시 전체 데이터 범위에 맞도록 지도 범위를 조정한다.
- 사용자가 필터나 검색을 바꾸면 현재 결과 범위 기준으로 다시 보기 쉽게 유지한다.
- "지도 맞춤" 동작은 현재 필터 결과 전체를 화면에 들어오게 맞춘다.

## 6. 팝업 정보

마커 팝업에는 다음 항목을 표시한다.

- 장소명
- 유형명
- 주소
- 전화번호
- 위도 / 경도

## 7. 예외 처리

- 자동 fetch 실패 시 실패 메시지를 노출한다.
- 폴백 로드에서도 서울 JSON 파일이 없으면 적절한 오류를 표시한다.
- 검색 또는 필터 결과가 0개면 빈 상태 안내를 보여준다.
- 좌표가 비정상적인 데이터는 제외하고, 전체 통계에는 좌표 포함 건수로만 집계한다.

## 8. 비기능 요구사항

- 단일 `index.html`에서 동작해야 한다.
- 외부 라이브러리는 CDN 기반으로 사용한다.
- 모바일 화면에서도 사이드바와 지도가 세로로 자연스럽게 재배치되어야 한다.
- 초기 화면은 로딩 상태를 명확히 보여주어야 한다.

## 9. 완료 기준

- 7개 서울 JSON 파일의 좌표가 지도에 모두 표시된다.
- 검색, 유형 필터, 목록 이동, 마커 팝업이 모두 정상 동작한다.
- 자동 로드 실패 시에도 폴백 로드로 데이터를 다시 불러올 수 있다.
